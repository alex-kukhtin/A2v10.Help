Робота с повідомленнями

Для того, щоб отделить логику построения повідомлення и механизм його отправки використовується механизм шаблонов. Шаблон являє собою набор правил, при помощи которых строится тіло повідомлення, його тема, адресаты, копии и т.д.

Шаблон може включать несколько сообщений. Доступ к ним осуществляется по простому текстовому ключу.

При построении повідомлення може використовуватися модель даних, додаткові параметри, які передаються отправителем и переменные окружения.

Когда системе потрібно отправить сообщение, она просто указывает, по какому шаблону його потрібно построить, и передает нужные параметри, такі как ім'яшаблона, ключ повідомлення, идентификатор для моделі даних и произвольные параметри.

Подсистема оправки выбирает потрібное сообщение, загружает модель даних (якщо потрібно), формирует непосредственно текст повідомлення з використанням привязок и отправляет його адресату.

Использование

Сообщение можна отправить следующими способами:

  • Из бізнес-процесу за допомогою активностей Request или SendMessage.
  • Непосредственно серверной командой типа sendMessage

Налаштування

  • Налаштування конфигурації.

Об'єкты для роботи с повідомленнями

  • EmailMessage - сообщение электронной почты.
  • MessageAddress - адрес повідомлення.
  • MessageAttachment - вложение.
  • MessageReport - звіт.
  • MessageReportModel - модель даних для звіта.
  • MessageReportParam - параметр моделі даних для звіта.
  • Template - шаблон.
  • TemplatedMessage - базовый класс для роботи с повідомленнями.
