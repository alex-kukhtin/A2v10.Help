Звіти (reports)

Під звітом в системі А2v10 мається на увазі або зовнішній звіт (підтримується звітна система компанії Stimulsoft), або файл, що вивантажується в форматі xml або json.

Об'єкт reports описує звіти, можливі в поточній кінцевій точці. Імена властивостей об'єкта являють собою імена звітів. Кожна властивість являє собою об`єкт наступної структури:

Властивості об'єкта звіту

Им'я Тип Опис
source string Не обов'язково. Джерело даних. Якщо не вказано, береться з батьківського елементу.
schema string Не обов'язково. Им'я sql-схеми. Якщо не вказано, береться з батьківського елементу.
model string Им'я моделі. Якщо не вказано, береться з батьківського елементу.
procedure string Им'я збереженої процедури для побудови звіту. Якщо не вказано, будується з найменування моделі з суфіксом '.Report'".
parameters object Об'єкт, статичні параметри, які передаються в збережену процедуру завантаження моделі. Им'я властивості є ім'ям параметра, значення - значенням параметра.
type enum Тип звіту. Можливі значения
  • stimulsoft - за замовчанням. Отчет stimulsoft. Повинно бути вказано значення властивості report;
  • xml - звіт в форматі xml;
  • json - звіт в форматі json.
Зверніть увагу. Звіти в форматі json часто використовуються в процессе відладки для того, щоб пересвідчитися в тому, що модель даних для звіту в форматі xml формується вірно.
report string Тільки для звітів stimulsoft. Им'я файла звіту без розширення. Буде додано розширення .mrt.
name string Им'я (заголовок) звіта. В режимі експорту це ім'я задає ім'я файлу, що експортується. Можна використовувати макроси. Деталі - нижче.
encoding string Тільки для xml-звітів. Кодування xml-файла. Являє собою будь-яке допустиме кодування. Частіше за все використовується 'utf-8', 'utf-16' або 'windows-1251'.
xmlSchemas string[] Тільки для xml-звітів. Массив рядків, які задають xml-схеми для побудови/перевірки xml-звітів. Імена файлів схем задаються без розширення (буде добавлено розширення .xsd) і відносно поточної папки (можна використовувати ../ для звернення до батьківської папки.)
validate boolean Тільки для xml-звітів. Чи перевіряти звіт на відповідність xml-схемам, що задані в масиві xmlSchemas.
variables object Об'єкт, який буде переданий у звіт stimulsoft в якості Variables. Використовується якщо потрібно змінити поведінку форми звіту в залежності від зовнішніх умов.

Детальніше про макроси

В імені файлу звіта можна використовувати макроси, які дозволяють звертатися до властивості поточної моделі. Макрос являє собою повне ім'я властивості (від кореневого елементу) в моделі даних у подвійних фігурних дужках.
