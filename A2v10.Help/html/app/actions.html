Дії (actions)

model.json

Об'єкт actions описує дії, можливі в поточній кінцевій точці. Імена властивостей об'єкта являють собою імена дій. Кожна властивість являє собою об`єкт наступної структури:

Властивості об'єкта дії

Ім'я Тип Опис
source string Не обов'язково. Джерело даних. Якщо не вказано, береться з батьківського елементу.
schema string Не обов'язково. Ім'я sql-схеми. Якщо не вказано, береться з батьківського елементу.
model string Ім'я моделі. Якщо не вказано, береться з батьківського елементу.
merge object Не обов'язково. Додаткова модель, яка буде об'єднана з основною. Можна задати іншу схему та інше джерело даних. Якщо схема або джерело не вказані, ввони беруться з батьківського елементу.
Об'єкт parameters дозволяє задавати додаткові параметри для другої моделі. Є можливість посилань на значення з основної моделі. Для цього вкажіть у значенні шлях до властивості основної моделі у подвійних фігурних дужках ({{....}}).
index boolean Чи є модель для дії індексною. Для індексної моделі виконується збережена процедура з суфіксом '.Index', для звичайної - з суфіксом '.Load'.
copy boolean Чи є дія дією копіювання. У цьому випадку для завантаження моделі виконується збережена процедура з суфіксом '.Copy'. Копіювання в індексних діях не підтримується.
template string Шлях до файлу шаблону для дії. Детальніше про шаблони.
view string Шлях до файлу виду для дії.
checkTypes string Ім'я файла *.d.ts для динамічної перевірки типів.
indirect boolean Непрямість. Дія виконується в два етапи.
parameters object Об'єкт, статичні параметри, які передаються в збережену процедуру завантаження моделі. Ім'я властивості є ім'ям параметру, значення - значенням параметру.
export object Об'єкт, що описує дію експорту (завантаження) моделі даних. Об'єкт має наступні параметри:
fileName Рядок. Ім'я файлу, що зберігається, яке буде запропоновано користувачу.
format Перелік xlsx | dbf | csv. Формат файлу для експорту.
template Рядок. Ім'я файлу шаблона, на підставі якого буде виконуватися експорт. Обов'язково для експорту у формат xlsx.
encoding Кодова сторінка файлу експорту. Обов'язково для експорту у формати dbf і csv. Перелік: utf8 | 1251 | 866.
При завантаженні моделі для експорту буде викликатися збережена процедура з суфіксом .Export.
