Файл model.json

Каждая папка прикладного приложения, описывающая конечную точку должна содержать файл описания model.json. В этом файле описываются все возможные действия, допустимые с этой конечной точкой.

Все объектные свойства представлены объектами, в которых имя свойства определяет название действия, а содержимое - объект, описывающий это действие.

Свойства объекта конечной точки

Имя Тип Описание
source String Источник данных. Фактически это имя строки подключения (connectionString) из файла конфигурации приложения. Если не указано - считается равным строке "Default". Если строка подключения не найдена - выбрасывается исключение
schema String Имя sql-схемы для модели данных. Может переопределяться в конкретных действиях. Если не указано - то dbo
model String Имя модели. Может переопределяться в конкретных действиях
actions actions Объект, описывающий возможные действия с этой конечной точкой
dialogs dialogs Объект, описывающий диалоги, которые можно вызывать для этой конечной точки
popups popups Объект, описывающий всплывающие окна, которые можно вызывать для этой конечной точки.
commands commands Объект, описывающий команды, которые можно выполнять для этой конечной точки.
reports reports Объект, описывающий отчеты, связанные с этой конечной точкой.
files files Объект, описывающий работу с загружаемыми файлами.
