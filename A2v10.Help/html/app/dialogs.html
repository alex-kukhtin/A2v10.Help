Діалоги (dialogs)

model.json

Об'єкт dialogs описує діалоги, можливі в поточній кінцевій точці. Імена властивостей об'єкта являють собою імена діалогів. Кожна властивість являє собою об`єкт наступної структури:

Властивості об'єкта діалога

Ім'я Тип Опис
source string Не обов'язково. Джерело даних. Якщо не вказано, береться з батьківського елементу.
schema string Не обов'язково. Ім'я sql-схеми. Якщо не вказано, береться з батьківського елементу.
model string Ім'я моделі. Якщо не вказано, береться з батьківського елементу.
merge object Не обов'язково. Додаткова модель, яка буде об'єднана с основною. Можна задати іншу схему і інше джерело даних. Якщо схема або джерело не вказані, вони берутся з батьківського елементу.
Об'єкт parameters дозволяє задавати додаткові параметри для другої моделі. Є можливість посилань на значення з основної моделі. Для цього вкажіть в значенні шлях до властивості основної моделі у подвійних фігурних дужках ({{....}}).
index boolean Чи показує діалог індексну модель. Для індексних моделей виконується збережена процедура з суфіксом '.Index', для звичайних - з суфіксом '.Load'.
copy boolean Чи є діалог діалогом копіювання. В цьому випадку виконується збережена процедура с суфіксом '.Copy'. Копіювання в індексних діалогах не підтримується.
template string Шлях до файлу шаблона для діалога. Детальніше про шаблони.
view string Шлях до файлу виду для діалога.
checkTypes string Ім'я файлу *.d.ts для динамічної перевірки типів.
parameters object Об'єкт, статичні параметри, які передаються в збережену процедуру завантаження моделі. Ім'я властивості є ім'ям параметра, значення - значенням параметра.
