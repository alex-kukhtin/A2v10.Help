Иерархические моделі даних

Моделі даних

Иерархическая (древовидная) модель даних включает набір записей c соотношением родитель-потомок. Глубина вложенности не ограничена.

Иерархическая модель представлена типом об'єкта !Tree. Для правильной роботи дерева каждая запись повинна мати три обязательных поля:

  • !Id - ідентифікатор записи
  • !ParentId - ідентифікатор родительской записи.
  • !Items - масив дочірніх полей.

Зверніть увагу. Для формирования набіров даних обычно використовується механизм рекурсивных запитов T-SQL (CTE) - Common Table Expressions) и оператор WITH. Подробнее в документации Microsoft.

Деревья можуть загружаться як статически (все дерево за однин запит), так і динамически (lazy loading), когда ветви дерева заполняются з бази даних тільки в том случае, якщо користувач (або программа) раскрывает эту ветку.

Статическое дерево

Статическое дерево заповнюється сразу повністю за одно обращение к БД. Запрос, формирующий дані для моделі дерева буде выглядеть наступним чином

При обработке цього набіра створюється простий масив Agentsелементів типа TAgent. В каждом елементе цього масива буде масив Items, который буде заполнен дочірніми елементуми. Родительский елементу визначається властивістю c модифікатором ParentId. Зверніть увагу, что это поле не має імені властивості, поэтому оно не попадет в результирующую модель.

Динамическое дерево

Динамическое дерево заповнюється по мере того, як користувач открывает определенные ветви дерева.

При роботі с динамическим деревом достаточно повернути з процедури завантаження тільки верхний уровень дерева. Однако для того, щоб система знала, какие елементи можна раскрыть, потрібно мати в моделі еще одно служебное властивість с специальным типом !HasChildren. Якщо такое властивість установлено, то система считает, що цет елемент має дочірні і показывает соответствующие елементи пользовательского інтерфейса.

Зверніть увагу, что такое властивість, несмотря на то, що оно служебное, обов'язково повинно мати имя, поскольку работа з ним ведется на клиентской стороне.

Коли користувач намагається раскрыть такой елемент дерева, викликаєся збережена процедура з суфіксом .Expand, яка повинна повернути дочірні елементи для цього узла. Далее процесс повторяется рекурсивно.
