Наборы данных

Первое поле каждого набора данных всегда должно иметь значение null. Имя этого поля определяет то, как интерпретировать этот набор данных.

Соглашения об именах полей

Любое имя в системе может состоять из нескольких (от одного до трех) элементов. Элементы имени различаются символом ! (восклицательный знак). Обратите внимание, что в SQL-Server имена, которые содержат спецсимволы, пробелы или ключевые слова языка нужно заключать в квадратные скобки. Все части имени являются необязательными, однако разделители обязательны всегда. Другими словами, если, например, имя поля начинается с !!, то это означает, что не указано имя (поля не будет в модели), нет типа, а есть только модификатор.

  • Первый элемент наименования - всегда имя свойства (возможно составное) в модели.
  • Второй элемент - имя типа либо путь к свойству в модели.
  • Третий элемент - модификатор, который описывает либо тип поля либо его специальное назначение.

Типы наборов данных (recordset)

Тип набора описывается третим элементом (суффиксом) первого поля каждого набора данных.

Тип Описание
Object Один элемент.
Array Массив элементов. Если этот массив дочерний, то в наборе должно присутствовать свойство, определяющее родительский элемент (!ParentId). Более подробно...
LazyArray "Ленивый" массив дочерних элементов. Загружается при первом обращении к свойству. Более подробно...
Map Ссылки. Всегда дочерний. Может не иметь имени. В наборе всегда должен быть идентификатор , (свойство с модификатором !Id). Именно по этому полю осуществляется связывание с источником.
Tree Дерево элементов (статическое или динамическое). В наборе обязательно должны присутствовать свойства с типами !Id, !ParentId, !Items. Более подробно...
Group Дерево элементов, которое строится с помощью группировки плоской таблицы. В наборе обязательно должны присутствовать свойства с типами !GroupMarker и !Items. Более подробно...
CrossArray Перекрестный массив элементов. Всегда дочерний. В наборе должно присутствовать свойство, определяющее родительский элемент (!ParentId) и свойство ключа !Key. Более подробно...
