Моделі с группировкой

Моделі даних

Модель даних с группировкой включает набір записей c соотношением родитель-потомок. Глубина вложенности не ограничена.

В отличие від ієрархічних (древовидных) моделей, в которых отношения родитель-потомок визначаються за допомогою идентификатора родительской записи, в моделях с группировкой отношения визначаються на основе повторяющихся значений.

Зверніть увагу. Для формирования набіров даних с группировкой обычно використовується оператор GROUP BY ROLLUP языка T-SQL. Подробнее в документации Microsoft.

Модель с группировкой представлена типом об'єкта !Group. Для правильной каждая запись повинна мати два додаткових обязательных поля:

  • !GroupMarker - маркеры группировки. Их буде стільки, сколько уровней группировки буде використано. Фактично являють собою просто значення T-SQL функції grouping для соответствующйого поля. Таких полей буде стільки же, сколько указано в операторе groupby rollup
  • !Items - масив дочірніх записей. В набірі даних це значення завжди буде дорівнює null. Поле необхідно для того, щоб система смогла побудувати иерархию исходя з полученных даних. Зверніть увагу, що тип даних елементів завжди повинен совпадать с типом даних кореневого об'єкта (с модифікатором !Group).

Из за особенностей обробки моделей НЕОБХОДИМО сначала отсортировать набір даних по маркерам группировки в обратном порядке, так щоб сначала выдавались группы.

Зверніть увагу. Якщо для моделі задан тільки один уровень группировки, то фактически это означает що вы просто подсчитаете итоги для определенных полей (тех, які выводятся за допомогою функції sum).

Приклад

Работу с моделями с группировкой проще всього продемострировать на примере. Нехай имееются таблиця Documents следующйого вида:

Id Date Agent Amount
10 2021-05-01 10 150.00
11 2021-05-02 10 300.00
12 2021-05-01 20 320.00
13 2021-05-02 20 270.00

И таблиця Agents следующйого вида:

Id Name
10 Agent 1
20 Agent 2

Построим набір даних с группировкой по дате і контрагенту.

В результаті роботи этой процедури сформується такой набір даних:

ReportData!TData!Group Agent Date Amount Agent!!GroupMarker Date!!GroupMarker Items!TData!Items
null null null 1040.00 1 1 null
null Agent 1 null 450.00 0 1 null
null Agent 2 null 590.00 0 1 null
null Agent 1 01.05.2020 150.00 0 0 null
null Agent 1 02.05.2020 300.00 0 0 null
null Agent 2 01.05.2020 320.00 0 0 null
null Agent 2 02.05.2020 270.00 0 0 null

Первые три строки набіра являють собою итоги по всему звіту (первая строка) і по каждому з контрагентів (вторая і третья строки). Маркеры группировки показывают для какого поля цей итог.

Після обробки цього набіра сформується такая модель (служебные властивості для простоты не показаны).

Зверніть увагу на властивості $level і $groupName. Властивість $level - визначає уровень в дереве (с 0), а $groupName - ім'ягруппы - фактически просто выбирается одно з значений набіра в залежності від уровня.
