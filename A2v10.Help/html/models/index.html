Модели данных

Модели данных - основные элемены из которых состоит бизнес-приложение. Модель это представление бизнес сущности, такой как, например клиент, поставщик или документ. Следует отметить, что модель это именно бизнес-сущность и она абстрагирована от таблиц БД и прочих технических подробностей. Более того, система вообще ничего не знает о существовании таблиц, именах полей и прочем. Она просто вызывает хранимые процедуры, которые должны вернуть/получить данные в оговоренном формате.

Фактически модель представляет собой иерархический объект, состоящий из скалярных значений, других объектов и массивов.

Система может выполнять с моделью всего четыре действия.

  • Получить список экземпляров.
  • Получить один экземпляр.
  • Сохранить экземпляр сущности.
  • Выполнить команду.

Для построения модели (и ее метаданных) платформа вызывает определенные хранимые процедуры. Их имена контсруируются из названия модели путем добавления специальных суффиксов к имени. Возможные суффиксы моделей:

  • .Index - загрузить список экземпляров.
  • .Load - загрузить один экземпляр.
  • .Metadata - загрузить метаданные для сохранения.
  • .Update - обновить экземпляр сущности.

Хранимые процедуры возвращают данные в виде нескольких наборов (recordset). Для построения метаданных используются имена полей и некоторые служебные поля и служебные наборы. Использование имен полей (а не их значений) необходимо потому, что нужно иметь возможность построить модели правильной струкруты даже из пустых наборов.

Виды моделей данных

  • Общая информация.
  • Простые модели данных.
  • Работа с массивами.
  • Разбиение на страницы.
  • Обновление моделей.
  • Иерархические модели.
  • Перекрестные модели.
  • Модели c группировкой.
  • Работа с двоичными объектами.
