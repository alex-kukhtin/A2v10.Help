Шаблоны. Опции

Опции шаблона задают поведение всей модели.

Синтаксис

Свойства объекта options

  • noDirty - не взводить флаг изменений $isDirty при изменениях модели данных.
  • bindOnce - массив строк, определяющий какие корневые свойства модели будут связываться однократно.
  • persistSelect - массив строк, определяющий массивы модели, для которых нужно сохранять позицию выделенного элемента в процессе сохранения или перезагрузки модели.
  • skipDirty - массив строк, определяющий пути к свойствам модели, при изменении которых не нужно взводить флаг изменений $isDirty.

Подробнее

Флаг $isDirty отвечает за вывод сообщения о необходимости сохранить модель после внесения изменений, а также за поведение некоторых элементов управления. Например кнопка, связанная с командой Save будет автоматически разрешаться после изменения модели. Если опция noDirty установлена, то флаг $isDirty взводиться не будет.

Свойство bindOnce позволяет определить корневые свойства модели, которые будут связываться с данными только при первой загрузке модели. При дальнейшей перезагрузке (например командой Reload или после сохранения модели) перепривязки этого свойства не будет. Чаще всего это используется для привязки элементов справочников.

Обратите внимание! В массиве bindOnce можно указывать только корневые свойства модели (то есть только те, в именах которых нет точки).

Свойство persistSelect позволяет определить массивы в модели, для которых будет сохраняться позиция выделенного элемента ($selected). Позиция будет сохраняться при сохранении модели (например командой Save) и при ее перезагрузке (например командой Reload).
