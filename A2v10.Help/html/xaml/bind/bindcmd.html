Прив'язка BindCmd

Все привязки

Представляет собой специальный об'єкт, що описує команду, яка буде выполнена в ответ на какое-то действие користувача (наприклад нажатие экранной кнопки). Практически всегда задається как розширення розмітки.

Зверніть увагу, что об`єкт має довольно много властивостей, но многие з них використовуються тільки для какой-то конкретной команди.

Властивості

Им'я Тип Опис
Command Command За замовчанням. Выполняемая команда. Можливі команди описаны ниже.
Argument Object Аргумент команди. Практически всегда привязка (Bind).
Data Object Дополнительные данные для команди. Практически всегда привязка (Bind).
Url String Url для внешней команди.
CommandName String Тільки для команди Execute. Им'я команди в шаблоне (template.commands).
SaveRequired Boolean Сохранять модель перед виконанням команди.
ValidRequired Boolean Для виконання команди требуется, щоб модель была в валидном состоянии.
Confirm Confirm Спрашивать у користувача подтверждение перед виконанням команди.
NewWindow Boolean Открывать в новом окне.
UpdateAfter Object Всегда привязка. Об'єкт або массив, который потрібно обновить после виконання как-либо дій.
ReloadAfter Boolean Обновить модель после виконання команди.
Export Boolean Выполнять ли експорт вместо отображения. Не совместима с Print.
Print Boolean Выполнять ли печать вместо отображения.
FileAction Enum Дія для команди File.
Action Enum Дія для команди Dialog.

Команди

Open - открыть форму

Выполняет переход на заданный Url с заданным аргументом.

Аргументы:

  • Url - Путь к форме. Обов'язково
  • Argument - Аргумент команди. Обов'язково.
  • NewWindow - Открывать в новом окне.
  • UpdateAfter - Прив'язка к елементу, который возможна буде обновляться при выполнении каких або дій. Имеет смысл тільки якщо форма открывается в новом окне.
Clear - очистить

Очищает об`єкт агрумента.

Аргументы:

  • Argument - Прив'язка к массиву або об'єкту, который потрібно очистить. Фактично команда просто викликає метод $empty связанного об'єкта або масива.

Зверніть увагу, что елемент керування, связанный з цією командой не буде выводиться, якщо об`єкт або массив уже пусты. Додаткова привязка властивості If не требуется.

Close - закрыть

Закрывает текущую форму або діалог. Команда не має аргументов.

CloseOk - закрыть

Тільки для діалогів. Закрывает текущий діалог с ответом true. Команда не має аргументов.

Reload, Refresh - обновить

Обновляет текущую модель або ее часть. Якщо аргумент задан, то обновляется тільки часть моделі. Обновление части моделі використовується для "ленивых массивов".

Аргументы:

  • Argument - Прив'язка к массиву, который потрібно обновить. Не обов'язково.
Requery - обновить

Перезагружает форму полностью с сервера (включая перегенерацию розмітки). Команда не має аргументов.

Print - напечатать

Печать поточної страницы. Команда не має аргументов. Зверніть увагу, что фактически печатается текущее изображение страницы. Неякі елементи (наприклад Taskpad) на печать не выводятся. Более тонко управлять видимостью елементів при печати можна за допомогою властивості Print об'єкта UIElement.

Dialog - діалог

Показывает діалог и виконує указанное действие.

Аргументы:

  • Action - действие. Можливі значения:
    • Edit - редактировать елемент, указанный аргументом. После закрытия діалога елемент буде обновлен.
    • EditSelected - редактировать выбранный елемент коллекции. Коллекция задається аргументом команди. После закрытия діалога выделенный елемент буде обновлен.
    • Show - показать елемент, заданный аргументом.
    • ShowSelected - показать выбранный елемент коллекции. Коллекция задається аргументом команди.
    • Browse - выбрать елемент, заданный аргументом. Елемент заменяется выбранным в діалоге.
    • Append - создать новый елемент и добавить його в коллекцию. Коллекция задається аргументом.
    • New - создать новый елемент.
    • Copy - создать копию заданного елементу. Елемент задається аргументом.
  • Argument - Прив'язка к елементу або коллекции.
  • Data - Дополнительные данные для вызова діалога. Будут параметрами процедури завантаження моделі діалога.
  • URL - URL на діалог з розділу dialogs файла описания моделі model.json.
  • SaveRequired, ValidRequired, ReloadAfter - додаткові условия команди.
File - работа с файлами (двоичными об'єктами)

Выполняет отображение, загрузку або печать вложения (двоичного об'єкта). Это може быть картика, файл и т.д. Детальніше про работе с двоичными об'єктами.

Аргументы:

  • FileAction - действие з файлом. Можливі значения:
    • Show - Показать данные на странице.
    • Download - завантажити данные как файл.
    • Print - Печать даних файла.
  • Argument - Прив'язка к властивості, которое буде задавати идентификатор двоичного об'єкта.
  • URL - URL на действие з розділу files файла описания моделі model.json.
Download - завантажити

Позволяет користувачу завантажити статический файл. Статические файли, пов'язані с приложением должны быть размещены в специальной папке _files.

Аргументы:

  • Url - шлях до статическому файлу. Путь зазначається відносно папки _files.
