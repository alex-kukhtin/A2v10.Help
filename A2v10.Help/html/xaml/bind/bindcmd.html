Прив'язка BindCmd

Все прив`язки

Являє собою специальный об'єкт, що описує команду, яка буде выполнена в ответ на какое-то дію користувача (наприклад нажатие экранной кнопки). Практически завжди задається як розширення розмітки.

Зверніть увагу, що об`єкт має довольно много властивостей, но многие з них використовуються тільки для какой-то конкретной команди.

Властивості

Ім'я Тип Опис
Command Command За замовчанням. Выполняемая команда. Можливі команди описаны ниже.
Argument Object Аргумент команди. Практически завжди прив'язка (Bind).
Data Object Додаткові дані для команди. Практически завжди прив'язка (Bind).
Url String Url для внешней команди.
CommandName String Тільки для команди Execute. Ім'я команди в шаблоне (template.commands).
SaveRequired Boolean Сохранять модель перед виконанням команди.
ValidRequired Boolean Для виконання команди требуется, щоб модель была в валидном состоянии.
Confirm Confirm Спрашивать у користувача подтверждение перед виконанням команди.
NewWindow Boolean Открывать в новом окне.
UpdateAfter Object Завжди привязка. Об'єкт або масив, который потрібно оновити після виконання как-либо дій.
ReloadAfter Boolean Обновить модель після виконання команди.
Export Boolean Выполнять ли експорт вмісце відображення. Не совместима с Print.
Print Boolean Выполнять ли печать вмісце відображення.
FileAction Enum Дія для команди File.
Action Enum Дія для команди Dialog.

Команди

Open - открыть форму

Виконує переход на заданий Url с заданным аргументом.

Аргументи:

  • Url - Путь к форме. Обов'язково
  • Argument - Аргумент команди. Обов'язково.
  • NewWindow - Открывать в новом окне.
  • UpdateAfter - Прив'язка к елементу, который возможна буде обновляться при виконанні каких або дій. Имеет смысл тільки якщо форма открывается в новом окне.
Clear - очистить

Очищает об`єкт агрумента.

Аргументи:

  • Argument - Прив'язка к масиву або об'єкту, который потрібно очистить. Фактично команда просто викликає метод $empty связанного об'єкта або масива.

Зверніть увагу, що елемент керування, связанный з цією командой не буде выводиться, якщо об`єкт або масив уже пусты. Додаткова прив'язка властивості If не требуется.

Close - закрыть

Закриває текущую форму або діалог. Команда не має аргументов.

CloseOk - закрыть

Тільки для діалогів. Закриває поточний діалог с ответом true. Команда не має аргументов.

Reload, Refresh - обновить

Обновляет текущую модель або її частину. Якщо аргумент задан, то обновляется тільки частина моделі. Оновлення части моделі використовується для "лінивих масивів".

Аргументи:

  • Argument - Прив'язка к масиву, который потрібно обновить. Не обов'язково.
Requery - обновить

Перезавантажує форму повністю с сервера (включая перегенерацию розмітки). Команда не має аргументов.

Print - напечатать

Печать поточної страницы. Команда не має аргументов. Зверніть увагу, що фактично печатается текущее зображення страницы. Деякі елементи (наприклад Taskpad) на печать не выводятся. Более тонко управлять видимостью елементів при печати можна за допомогою властивості Print об'єкта UIElement.

Dialog - діалог

Показывает діалог і виконує указанное действие.

Аргументи:

  • Action - действие. Можливі значення:
    • Edit - редактировать елемент, указанный аргументом. Після закриття діалога елемент буде обновлен.
    • EditSelected - редактировать выбранный елемент коллекции. Колекція задається аргументом команди. Після закриття діалога виділений елемент буде обновлен.
    • Show - показать елемент, заданий аргументом.
    • ShowSelected - показать выбранный елемент коллекции. Колекція задається аргументом команди.
    • Browse - выбрать елемент, заданий аргументом. Елемент заменяется выбранным в діалоге.
    • Append - создать новий елемент і додати його в коллекцию. Колекція задається аргументом.
    • New - создать новий елемент.
    • Copy - создать копию заданного елементу. Елемент задається аргументом.
  • Argument - Прив'язка к елементу або коллекции.
  • Data - Додаткові дані для виклику діалога. Будут параметрами процедури завантаження моделі діалога.
  • URL - URL на діалог з розділу dialogs файла описания моделі model.json.
  • SaveRequired, ValidRequired, ReloadAfter - додаткові условия команди.
File - работа с файлами (бінарними об'єктами)

Виконує отображение, загрузку або печать вкладення (двоичного об'єкта). Це може быть картика, файл і т.д. Детальніше про роботі с бінарними об'єктами.

Аргументи:

  • FileAction - дію з файлом. Можливі значення:
    • Show - Показать дані на странице.
    • Download - завантажити дані як файл.
    • Print - Печать даних файла.
  • Argument - Прив'язка к властивості, которое буде задавати ідентифікатор двоичного об'єкта.
  • URL - URL на дію з розділу files файла описания моделі model.json.
Download - завантажити

Позволяет користувачу завантажити статический файл. Статические файли, пов'язані с приложением мають бути розміщені в специальной папке _files.

Аргументи:

  • Url - шлях до статическому файлу. Путь зазначається відносно папки _files.
