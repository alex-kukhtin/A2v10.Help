Робота с повідомленнями

Для того, щоб відділити логіку побудови повідомлення і механизм його відправки використовується механизм шаблонов. Шаблон являє собою набір правил, за допомогою яких будується тіло повідомлення, його тема, адресати, копії і т.д.

Шаблон може включати декілька повідомлень. Доступ до них відбувається по простому текстовому ключу.

При побудові повідомлення може використовуватися модель даних, додаткові параметри, які передаються відправником і змінні середовища.

Коли системі потрібно надіслати повідомлення, вона просто вказує, по якому шаблону його потрібно побудувати, і передає потрібні параметри, такі, як ім'я шаблону, ключ повідомлення, ідентифікатор для моделі даних і довільні параметри.

Підсистема відправки вибирає потрібне повідомлення, завантажує модель даних (якщо потрібно), формує безпосередньо текст повідомлення з використанням привязок і надсилає його адресату.

Використання

Повідомлення можна надіслати наступними способами:

  • З бізнес-процесу за допомогою активностей Request або SendMessage.
  • Безпосередньо серверною командою типу sendMessage

Налаштування

  • Налаштування конфигурації.

Об'єкти для роботи с повідомленнями

  • EmailMessage - повідомлення електронної пошти.
  • MessageAddress - адреса повідомлення.
  • MessageAttachment - вкладення.
  • MessageReport - звіт.
  • MessageReportModel - модель даних для звіта.
  • MessageReportParam - параметр моделі даних для звіта.
  • Template - шаблон.
  • TemplatedMessage - базовий клас для роботи с повідомленнями.
