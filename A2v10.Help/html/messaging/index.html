Робота с повідомленнями

Для того, щоб отделить логику построения повідомлення і механизм його отправки використовується механизм шаблонов. Шаблон являє собою набор правил, при помощи которых строится тіло повідомлення, його тема, адресаты, копии і т.д.

Шаблон може включать несколько сообщений. Доступ к ним осуществляется по простому текстовому ключу.

При построении повідомлення може використовуватися модель даних, додаткові параметри, які передаються отправителем і переменные окружения.

Когда системе потрібно отправить повідомлення, она просто указывает, по какому шаблону його потрібно построить, і передает нужные параметри, такі як ім'яшаблона, ключ повідомлення, ідентифікатор для моделі даних і произвольные параметри.

Подсистема оправки выбирает потрібное повідомлення, завантажує модель даних (якщо потрібно), формирует непосредственно текст повідомлення з використанням привязок и отправляет його адресату.

Использование

Повідомлення можна отправить наступними способами:

  • Из бізнес-процесу за допомогою активностей Request или SendMessage.
  • Непосредственно серверной командой типа sendMessage

Налаштування

  • Налаштування конфигурації.

Об'єкты для роботи с повідомленнями

  • EmailMessage - повідомлення электронной почты.
  • MessageAddress - адрес повідомлення.
  • MessageAttachment - вложение.
  • MessageReport - звіт.
  • MessageReportModel - модель даних для звіта.
  • MessageReportParam - параметр моделі даних для звіта.
  • Template - шаблон.
  • TemplatedMessage - базовый класс для роботи с повідомленнями.
